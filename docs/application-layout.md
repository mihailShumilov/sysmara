# SysMARA Application Layout

This document defines the canonical directory structure for applications built with SysMARA.

## Directory Structure

```
/app                        Application code
  /entities                 Entity implementations
  /capabilities             Capability handler implementations
  /policies                 Policy enforcement logic
  /invariants               Invariant validation logic
  /modules                  Module-level composition
  /flows                    Flow orchestration
  /routes                   Route definitions and middleware
  /services                 Shared application services
  /adapters                 External system adapters (DB, APIs, etc.)
  /generated                Compiler-generated code (DO NOT EDIT)
  /protected                Infrastructure code (REQUIRES AUTHORIZATION)
  /tests                    Application tests

/system                     Machine-readable specifications (YAML)
  architecture.yaml         System-level architecture metadata
  entities.yaml             Entity definitions
  capabilities.yaml         Capability definitions
  policies.yaml             Policy definitions
  invariants.yaml           Invariant definitions
  modules.yaml              Module and boundary definitions
  flows.yaml                Flow definitions
  safe-edit-zones.yaml      File ownership and edit permissions
  glossary.yaml             Domain glossary

/.framework                 Framework-generated artifacts (DO NOT EDIT)
  system-map.json           AI-facing system index
  system-graph.json         Full system graph
  diagnostics.json          Latest diagnostics report
  generated-manifest.json   Manifest of all generated files

sysmara.config.yaml           Framework configuration
```

## Folder Details

### `/app/entities`
- **Purpose:** TypeScript implementations of domain entities
- **AI editable:** Yes
- **Generated:** No (but scaffolds may be generated into /generated)
- **Owner:** Module owner

### `/app/capabilities`
- **Purpose:** Handler implementations for each capability
- **AI editable:** Yes
- **Generated:** No (stubs generated into /generated, move to here when implementing)
- **Owner:** Module owner

### `/app/policies`
- **Purpose:** Policy enforcement middleware and logic
- **AI editable:** Yes, with invariant awareness
- **Generated:** No
- **Owner:** Module owner

### `/app/invariants`
- **Purpose:** Runtime invariant validation functions
- **AI editable:** Yes, but must match spec definitions
- **Generated:** No
- **Owner:** Module owner

### `/app/modules`
- **Purpose:** Module composition and wiring
- **AI editable:** Yes
- **Generated:** No
- **Owner:** Module owner

### `/app/flows`
- **Purpose:** Multi-step flow orchestration
- **AI editable:** Yes
- **Generated:** No
- **Owner:** Module owner

### `/app/routes`
- **Purpose:** HTTP route definitions connecting paths to capabilities
- **AI editable:** Yes
- **Generated:** Route stubs generated by compiler
- **Owner:** Module owner

### `/app/services`
- **Purpose:** Shared application services (logging, config, etc.)
- **AI editable:** Yes
- **Generated:** No
- **Owner:** Platform team

### `/app/adapters`
- **Purpose:** External system adapters (databases, third-party APIs)
- **AI editable:** Yes
- **Generated:** No
- **Owner:** Platform team

### `/app/generated`
- **Purpose:** All compiler-generated code
- **AI editable:** NO — will be overwritten on recompile
- **Generated:** Yes
- **Owner:** SysMARA Capability Compiler

### `/app/protected`
- **Purpose:** Infrastructure and security-critical code
- **AI editable:** NO — requires explicit authorization
- **Generated:** No
- **Owner:** Platform/security team

### `/app/tests`
- **Purpose:** Application test files
- **AI editable:** Yes (test scaffolds are generated as editable)
- **Generated:** Scaffolds only
- **Owner:** Module owner

### `/system`
- **Purpose:** YAML specifications — the source of system truth
- **AI editable:** Yes, with validation
- **Generated:** No
- **Owner:** System architect / AI agent (with review)

### `/.framework`
- **Purpose:** Framework output artifacts
- **AI editable:** NO — regenerated by framework commands
- **Generated:** Yes
- **Owner:** SysMARA framework
